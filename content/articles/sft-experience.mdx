# SFT 商品短标题微调经验

## 背景介绍

在做一个商品短标题生成微调任务，目标让 Qwen3 在生成效果上接近 Gemini。

做法是将 Gemini 产出的数据作为训练素材，也就是用 Gemini 生成的结果作为 SFT 的标准答案。

## 解决方案

一开始直接将自己给 Gemini 的 Prompt 作为 SFT 的 System prompt 进行训练，效果确实有提升。

后来经过与同行交流，意识到这个任务本质上是一个 **xy 映射问题**，不一定需要完整的原始 System prompt。

**优化后的做法：**

- 对 System prompt 进行压缩，训练出简短 prompt 到目标输出的映射关系
- 批量推理时使用压缩后的简短 System prompt
- 好处是推理速度更快，训练速度也更快

## 其他可能的解决方案

1. **保持完整 System prompt 不压缩**：如果推理资源充足、延迟不敏感，可以不考虑压缩。压缩本身有丢失信息的风险。
2. **用更小的模型做蒸馏**：不直接微调 Qwen3，而是用 Gemini 的输出先微调一个更小的模型，再蒸馏到 Qwen3，适合推理成本敏感的场景。
3. **用 DPO / RLHF 替代纯 SFT**：如果有多轮反馈数据，可以结合人类偏好对齐进一步提升生成质量，而不只是模仿 Gemini 的输出。
